# My Experience

Dieses Verzeichnis ist das zentrale Wissensledger fuer operative Erfahrung in diesem Repo.

## Zweck

- Wiederkehrende Learnings und Stolpersteine zentral erfassen.
- Erkenntnisse aus Features, Improvements und Defects verdichten.
- Vor grossen Fixes schnell auf bekannte Muster zugreifen.

## Update-Regeln

- Jede abgeschlossene Aufgabe aktualisiert sowohl den Kategorie-Eintrag unter `docs/my-*` als auch dieses zentrale Verzeichnis.
- `overview.md` kurz halten und nur relevante Kern-Learnings pflegen.
- `history.json` chronologisch erweitern (ISO-Zeitstempel, kurze Aktion).
- Schwere Bugs immer in `severe-bug-history.md` dokumentieren.
- Vor jedem groesseren Bugfix zuerst `severe-bug-history.md` pruefen.
